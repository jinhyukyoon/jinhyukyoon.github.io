1. Robot control software

My role: control algorithm/software development.
Skills: C++, linux RT, real-time control, teleoperation, EtherCAT, system integration, system debugging, validation

I led on control algorithms and software development for a minimally invasive laparoscopic surgical robot, Stark. Stark is a teleoperation system with master and slave robot that involves many constraints such as surgical instruments, anchored user experience from exsiting system like Da Vinci, and safety requirements as a medical device. As a founding member of the Robot R&D Department and Control/SW team, I participated development stages from prototype to engineering sample.
I translated system level requirements into control software behavior that could run on real system. This involved decomposing robot functions into software module and architecture, integrating hardware interfaces, debugging real-time behavior on the robot, and validating the system beyond isolated bench tests. The goal was to build a robot system that could be controlled, debugged, repeated, and trusted in realistic surgical workflows.


2. Long distance teleoperation

My role: teleoperation integration, system debugging, validation execution

I contributed to a long distance remote surgical procedure demonstration over more than 3,000 km with a measured round-trip delay of 65 ms. The work focused on keeping the whole robot system predictable when human input, network delay, robot motion, and hardware response were connected end to end.
I worked across master, slave, control software, and communication layer. Many issues only surfaced after the robot system and network were connected as a full teleoperation system. I traced problems from operator input to physical robot response, focusing on how latency and timing variations affected stability, responsiveness, and operator control. The demonstration validated the software within an end-to-end surgical workflow, where control behavior, system latency, hardware response, and human interaction had to be tested together.


3. System and software architecture

My role: system/software architecture design, requirement engineering, interface definition, software V&V

As the robot system and team grew, the main challenge shifted from implementation to system clarity. Important requirements were often hidden inside clinical expectations, business milestone, and different specialties of each team.
I turned those implicit requirements into architecture, specifications, interfaces, and validation items. The system had to be clear enough to build, test, review, and maintain across teams. As the medical device, the software architecture and V&V work also considered regulatories and safety standards such as the IEC 60601 family. Software behavior, hardware interfaces, alarms, failure handling, essential performance, and test evidence were treated as connected parts of the same system problem. I helped connect requirements to implementation and validation so that system behavior could be tested and justified, rather than only demonstrated. I also led software V&V with more than 10 software engineers and coordinated architecture and specification work across more than 40 cross-functional members.


4. Labs and demonstrations

My role: system operation/debugging, experiment support, demo execution, issue triage

I led system operation for 6 animal labs, 5 cadaver labs, and more than 20 investor demonstrations for Series E, pre-IPO, and IPO milestones. These settings tested the robot as a complete system, while robot system was still under development with numerous improvements.
Labs and demonstrations tested the robot beyond ordinary development conditions. System setup, surgical workflow, hardware variation, timing, operator interaction, and subsystem dependencies all became part of the engineering problem. I kept the system operational, traced failures quickly, and brought the findings back into software, control, integration, and validation. Each run helped clarify requirements, strengthen tests, and improve implementation.
